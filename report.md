# v0.3.0 UI audit notes

- First screen needed a clearer product narrative, so Overview now leads with running status, Quick Connect, Setup Progress, core metrics, and a recommended next action.
- Cards and tables needed stronger visual hierarchy, so the dashboard now uses CSS variables, larger metrics, clearer badges, softer surfaces, and screenshot-ready page headers.
- Combo Models needed to stand out as the core value, so the page now shows the client model name, routing path, candidate details, and problem hints.
- Clients and Usage needed to be more directly usable, so client cards expose copy-ready configs and Usage keeps metrics, filters, and redacted request metadata visible.
- Diagnostics was too technical for the main flow, so redacted diagnostic summaries and advanced internal state are grouped behind clearer progressive disclosure.
